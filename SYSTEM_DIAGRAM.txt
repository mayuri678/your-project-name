    USER ACTION                    SYSTEM RESPONSE
    
    1. Click "Create Resume"  →    Open template selection
                                   ↓
    2. Browse templates       →    Display all 55 templates
                                   ↓
    3. Click Template 5       →    Navigate to:
                                   /resume-builder?templateId=5
                                   ↓
    4. Resume Builder loads   →    Read templateId from URL
                                   ↓
                              →    Query TemplateRegistry
                                   ↓
                              →    Get Template5Component
                                   ↓
                              →    Create component dynamically
                                   ↓
                              →    Render Template 5 layout
                                   ↓
    5. Fill in form data      →    Update resumeData object
                                   ↓
                              →    Pass data to template
                                   ↓
                              →    Template re-renders
                                   ↓
    6. Click "Save"           →    Send to Supabase:
                                   {
                                     user_id: "abc",
                                     template_id: 5,
                                     resume_data: {...}
                                   }
                                   ↓
    7. Click "Download PDF"   →    Generate PDF from template
                                   ↓
                              →    Download file

│                                                                             │

    ✅ Each template is a separate Angular component
    ✅ Templates are loaded dynamically based on user selection
    ✅ Template ID is passed via URL query parameter
    ✅ All templates share the same ResumeData interface
    ✅ Each template has unique HTML and CSS
    ✅ Template Registry maps IDs to components
    ✅ Supabase stores template selection with resume data
    ✅ ViewContainerRef enables dynamic component loading
    ✅ Change detection updates template when data changes
    ✅ Users can switch templates without losing data
